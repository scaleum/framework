# Hydrator

**Hydrator** — базовий клас для ініціалізації об'єктів з масиву конфігурації та створення екземплярів.

## Призначення

- Автоматична ініціалізація об'єктів з масиву даних
- Створення екземпляра класу з передачею параметрів конструктора
- Уніфікація ініціалізації об'єктів у проєкті

## Основні можливості

| Метод | Призначення |
|:------|:-----------|
| `__construct(array $config = [])` | Ініціалізація об'єкта за допомогою конфига |
| `createInstance(mixed $input)` | Створення екземпляра класу зі строки або масиву з конфігурацією |

## Приклади використання

### Ініціалізація об'єкта з конфігом

```php
$config = [
    'property1' => 'value1',
    'property2' => 'value2',
];

$object = new Hydrator($config);
```

### Створення екземпляра через `createInstance`

```php
$instance = Hydrator::createInstance([
    'class' => MyClass::class,
    'config' => ['param1' => 'value1', 'param2' => 'value2'],
]);
```

### Створення екземпляра за ім'ям класу

```php
$instance = Hydrator::createInstance(MyClass::class);
```

## Додатково

Клас використовує `InitTrait` для ініціалізації.

Метод `createInstance` автоматично:
- Перевіряє існування класу.
- Створює екземпляр, передаючи параметри конструктора.
- Якщо клас реалізує `HydratorInterface`, передає весь конфіг напряму.

## Винятки

- `RuntimeException` — якщо клас не існує.
- `ERuntimeError` — якщо відсутній обов'язковий параметр конструктора.
