# Gettext

[EN](../../../../en/components/i18n/loaders/Gettext.md) | **UK** | [RU](../../../../ru/components/i18n/loaders/Gettext.md)
Клас‑завантажувач перекладів у форматі GNU Gettext (`.po`). Наслідує `TranslationLoaderAbstract` і реалізує розбір файлу за допомогою регулярних виразів, формуючи колекцію перекладів у вигляді `ArrayObject`.

## Методи

| Підпис                                | Повертаємий тип | Доступ    | Призначення                                                                                                                                           |
| ------------------------------------- | --------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `load(string $filename): ArrayObject` | `ArrayObject`   | public    | Завантажує `.po`‑файл, проводить попередню валідацію через `validateFile`, розбирає пари `msgid`/`msgstr` і повертає їх у вигляді `ArrayObject`.       |
| `decode(string $str): string`         | `string`        | protected | Перетворює рядок із формату Gettext: прибирає лапки, екрановані символи (`\n`, `\t` тощо). Використовується всередині `load`.                         |

## Приклад використання

```php
<?php

use Scaleum\i18n\Loaders\Gettext;

$loader = new Gettext(); // змінна lowerCamelCase

$messages = $loader->load(__DIR__ . '/messages/ru.po');

// Вивід перекладу рядка "greeting"
echo $messages['greeting']; // Привіт
```

## Приклад `.po`‑файлу

```po
msgid "greeting"
msgstr "Привіт"

msgid "farewell"
msgstr "До побачення"
```

## Поради щодо застосування

* **Продуктивність**: якщо обсяг перекладів великий, обгорніть завантажувач кешем (наприклад, `PSR-16 Cache`) для уникнення повторного читання диска.
* **Валідність**: метод `validateFile` базового класу перевіряє існування та доступність файлу, але не його коректність. За потреби розширюйте перевірку, аналізуючи заголовки `.po`.
* **Локальне оточення**: переконайтеся, що файли `.po` зберігаються в UTF‑8 без BOM. Це запобіжить проблемам із кодуванням при розборі.
