# ModelData

`ModelData` — зручна «обгортка»‑контейнер над масивом атрибутів моделі. Наслідується від `AttributeContainer` і розташовується в просторі імен `Scaleum\Storages\PDO`. Клас виконує *натуралізацію* вхідного масиву (приводить ключі/значення до канонічного вигляду) і вміє рекурсивно перетворювати вкладені об’єкти/моделі в масив.

## Методи

| Підпис                                | Повертаємий тип | Доступ    | Призначення |
| ------------------------------------- | --------------- | --------- | ----------- |
| `__construct(array $attributes = [])` | —               | public    | Приймає початкові атрибути, пропускає їх через `ArrayHelper::naturalize()` і передає в базовий `AttributeContainer`.           |
| `toArray(): array`                    | `array`         | public    | Повертає масив атрибутів, попередньо нормалізуючи кожне значення через `normalizeValue()`.                                  |
| `isEmpty(): bool`                     | `bool`          | public    | Визначає, чи порожній контейнер (перевіряє кількість атрибутів через `getAttributeCount()`).                                 |
| `normalizeValue(mixed $value): mixed` | `mixed`         | protected | Рекурсивно нормалізує значення: розгортає вкладені `ModelData`, `ModelAbstract`, об’єкти з методом `toArray()` або масиви. |

## Алгоритм `normalizeValue()`

1. **Вкладений `ModelData`** → повертає `$value->toArray()`.
2. **Вкладена модель (`ModelAbstract`)** → використовує `$value->getData()->toArray()`.
3. **Об’єкт з методом `toArray()`** → викликає його.
4. **Масив** → застосовує нормалізацію до всіх елементів рекурсивно.
5. **Інші значення** → повертає «як є».

## Приклад використання

```php
<?php

declare(strict_types=1);

use Scaleum\Storages\PDO\ModelData;

// Вхідні дані (можуть містити підмасиви та об’єкти-моделі)
$raw = [
    'id'       => 42,
    'name'     => 'Alice',
    'settings' => [
        'notifications' => true,
    ],
];

$modelData = new ModelData($raw); // змінна lowerCamelCase

// Конвертація в масив (за потреби рекурсивно перетворює вкладені об’єкти)
print_r($modelData->toArray());

// Перевірка, що контейнер не порожній
if (! $modelData->isEmpty()) {
    // ...
}
```

## Практичні поради

* **Immutable‑підхід**: не змінюйте масив атрибутів напряму — використовуйте методи `AttributeContainer` (`setAttribute()`, `getAttribute()`) для контролю цілісності.
* **Вкладені структури**: якщо ваша модель містить вкладені моделі/об’єкти, `ModelData` автоматично «сплющить» їх, роблячи JSON‑серіалізацію простою.
* **Функціональне зберігання**: `ModelData` зручно зберігати як окрему властивість моделі (`getData()`), не змішуючи бізнес‑логіку з даними.
