#  Архітектура проєкту

##  Загальна концепція

Фреймворк побудований на принципах модульності, читабельності та суворого розділення відповідальності.  
Кожен компонент ізольований у своїй області (`namespace`) і виконує обмежену кількість завдань.


##  Структура вихідників фреймворку (`/src/`)

- `Cache/` — кешування
- `Config/` — конфігурація та завантаження параметрів
- `Console/` — реалізація ядра для роботи в режимі CONSOLE
- `Core/` — ядро фреймворку
- `DependencyInjection/` — контейнер залежностей (DI)
- `Events/` — подієва шина
- `Http/` — реалізація ядра для роботи в режимі HTTP
- `i18n/` — інтернаціоналізація
- `Logger/` — логування подій
- `Routing/` — маршрутизація та диспетчеризація
- `Security/` — безпека: автентифікація та авторизація
- `Services/` — менеджмент сервісів
- `Session/` — управління сесіями
- `Stdlib/` — допоміжні класи
- `Storages/` — драйвери для зберігання даних


##  Структура проєкту на основі фреймворку (приблизна)

- `/application/` — код самого застосунку, включно з:
  - `/commands/` — файли конфігурацій консольних команд
  - `/config/` — файли конфігурацій
  - `/migrations/` — файли міграцій
  - `/src/` — прикладні класи та сервіси проєкту
  - `/routes/` — файли конфігурацій роутингу
  - `/translations/` — файли локалізацій
  - `/log/` — файли журналів логів
  - `/views/` — шаблони подань
- `/public/` — публічна директорія (точка входу `index.php`, ассети)
- `/vendor/` — сторонні залежності, встановлені через Composer
